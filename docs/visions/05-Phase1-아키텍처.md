# Phase 1 아키텍처 결정 — Local-First 구조

> 작성일: 2026-03-01  
> 상태: 아키텍처 결정

---

## 핵심 결정

> **Phase 1에서는 중앙 서버를 두지 않는다.**  
> 모든 데이터 수집·처리·렌더링은 **사용자의 로컬 디바이스**에서 수행된다.

---

## 아키텍처 개요

### Local-First 구조

```
┌─────────────────────────────────────────────────────┐
│                 사용자 디바이스 (로컬)                  │
│                                                     │
│  ┌───────────────┐     ┌──────────────────────────┐ │
│  │ 데스크톱 앱     │────▶│  Local Engine (핵심 엔진)   │ │
│  │ (Tauri 등)     │     │                          │ │
│  └───────────────┘     │  ┌────────────────────┐  │ │
│                        │  │ 브라우저 자동화 모듈  │  │ │
│  ┌───────────────┐     │  │ (CDP / Playwright)  │  │ │
│  │ 모바일 앱      │────▶│  └────────────────────┘  │ │
│  │ (로컬 연동)    │     │                          │ │
│  └───────────────┘     │  ┌────────────────────┐  │ │
│                        │  │ LLM 연동 모듈       │  │ │
│                        │  │ (API Key: 사용자 소유)│  │ │
│                        │  └────────────────────┘  │ │
│                        │                          │ │
│                        │  ┌────────────────────┐  │ │
│                        │  │ Content Normalizer  │  │ │
│                        │  │ (콘텐츠 규격화)      │  │ │
│                        │  └────────────────────┘  │ │
│                        │                          │ │
│                        │  ┌────────────────────┐  │ │
│                        │  │ Local DB / Cache    │  │ │
│                        │  │ (SQLite 등)         │  │ │
│                        │  └────────────────────┘  │ │
│                        └──────────────────────────┘ │
│                                  │                  │
│                                  ▼                  │
│                        ┌──────────────────┐         │
│                        │ 사용자 브라우저    │         │
│                        │ (사용자 본인 계정)  │         │
│                        └──────────────────┘         │
└──────────────────────────────────────────────────────┘
                           │
                           ▼ (사용자 디바이스에서 직접)
                ┌─────────────────────┐
                │   외부 플랫폼들       │
                │  YouTube, Naver 등   │
                └─────────────────────┘
```

### 중앙 서버형과의 비교

| 항목 | 중앙 서버형 | Local-First (Phase 1 채택) |
|------|-----------|---------------------------|
| 데이터 수집 주체 | 운영자 서버 | 사용자 본인 디바이스 |
| 플랫폼 접근 주체 | 운영자 | 사용자 본인 (본인 계정/브라우저) |
| 데이터 저장 | 중앙 DB | 사용자 로컬 (SQLite 등) |
| LLM 비용 부담 | 운영자 | 사용자 (본인 API Key) |
| 법적 책임 소재 | **운영자에게 집중** | **사용자 개인 행위로 분산** |
| 설치 복잡도 | 낮음 (웹 접속) | 높음 (로컬 도구 설치 필요) |
| 확장성 | 높음 | 제한적 (디바이스 성능 의존) |
| 프라이버시 | 리스크 있음 | 우수 (데이터가 로컬에만 존재) |

## 구성 요소 상세

### 1) Local Engine (핵심 엔진)

- 사용자 디바이스에 설치되는 백그라운드 서비스
- **후보 기술**: Rust(Tauri sidecar), Node.js, Python
- 역할: 브라우저 자동화, LLM 호출, 콘텐츠 규격화, 로컬 캐싱

### 2) 브라우저 자동화 모듈

- 사용자 **본인의 브라우저**를 CDP(Chrome DevTools Protocol)로 제어
- 사용자가 이미 로그인된 세션을 활용 → 별도 인증 불필요
- **후보 기술**: Playwright, Puppeteer, Chrome CDP 직접 연결

### 3) LLM 연동 모듈

- 사용자가 **본인의 API Key**를 입력하여 사용
  - OpenAI, Anthropic, Google 등 선택 가능
  - 또는 로컬 LLM (Ollama 등) 지원
- 운영자 서버를 경유하지 않음

### 4) 프론트엔드 앱

- **데스크톱**: Tauri 기반 — 경량, 네이티브 성능, Rust 백엔드
- **모바일 연동**: 로컬 네트워크를 통해 데스크톱 엔진과 연동
  - 초기: 모바일 브라우저에서 로컬 웹 서버 접속
  - 이후: 네이티브 모바일 앱 (React Native / Flutter)

### 5) 로컬 데이터 저장

- SQLite: 규격화된 콘텐츠 캐시, 사용자 설정, 히스토리
- 사용자 데이터는 **절대로 외부 전송하지 않음**

## 설치 및 사용 흐름

```
1. 사용자가 데스크톱 앱 다운로드 및 설치 (Tauri 패키지)
       ↓
2. 초기 설정
   - LLM API Key 입력 (또는 로컬 LLM 선택)
   - 브라우저 연동 설정 (Chrome 경로 지정)
   - 선호 플랫폼/소스 설정
       ↓
3. Local Engine이 백그라운드로 실행
       ↓
4. 앱에서 자연어로 요청
   예: "최근 인기 있는 AI 관련 영상 보여줘"
       ↓
5. 로컬에서 모든 처리 수행
   - 사용자 브라우저로 유튜브 검색
   - 결과 규격화
   - LLM으로 분석/큐레이션
       ↓
6. 앱에서 맞춤형 UI로 결과 렌더링
```

## 모바일 연동 방식

### Phase 1: 로컬 네트워크 연동

```
[모바일 기기]  ──── 같은 Wi-Fi ────  [데스크톱 (Local Engine)]
     │                                      │
     │         HTTP / WebSocket             │
     └──────────────────────────────────────┘
```

- 데스크톱 엔진이 로컬 웹 서버 역할
- 모바일에서 브라우저 또는 앱으로 접속
- **장점**: 모바일에서도 데스크톱의 브라우저 자동화 능력 활용
- **제약**: 같은 네트워크 필요, 데스크톱이 켜져 있어야 함

### Phase 2+ (향후): 독립 모바일 앱

- 모바일 자체에서 경량 엔진 실행
- WebView 기반 브라우저 자동화
- 로컬 LLM (on-device) 활용 가능성
