## 1. LLM 활용 영역의 확장 — MCP의 등장

- **MCP (Model Context Protocol)** 등의 프로토콜로 LLM의 활용 영역이 비약적으로 향상됨
- LLM이 외부 도구, 데이터 소스, API에 표준화된 방식으로 접근 가능해짐
- 단순 텍스트 생성을 넘어 **실행 가능한 에이전트**로의 전환이 가속화

## 2. A2A의 등장과 MCP/A2A 상호보완 생태계

- **A2A (Agent-to-Agent)** 프로토콜 등장
- MCP: LLM ↔ 도구/데이터 연결 (수직적)
- A2A: 에이전트 ↔ 에이전트 연결 (수평적)
- 양 프로토콜이 **상호보완적 생태계**를 형성할 수 있다는 시각이 대두

## 3. AgentHub 프로젝트 (진행 중)

- **기반 기술**: Google ADK (Agent Development Kit)
- **아키텍처**: A2A 연결 + MCP 연결 → 백엔드 ↔ 브라우저 Extension 연결
- **성격**: 실험적 / 포트폴리오 목적의 프로젝트
- **핵심 기능**: 에이전트 허브를 통한 다양한 AI 에이전트의 통합 운영

## 4. OpenClaw — AI 운영체제 개념

- **OpenClaw**: 혁신적 오픈소스 프로젝트
- **핵심 개념**: AI의 운영체제(OS)
- LLM이 브라우저를 직접 제어 (CDP 모드 등 활용)
- 단순 스크래핑/검색이 아닌, **제한 없는 브라우저 직접 제어**
- 사용자의 자연어 지시만으로 복잡한 웹 작업 수행 가능

## 5. AgentHub의 한계 인식

| 항목 | 내용 |
|------|------|
| **프레임워크 종속** | Google ADK에 종속적 |
| **실행 환경 제약** | 백엔드 서버 실행 필수 |
| **배포 제약** | 브라우저 Extension을 개발자 모드로 설치해야 함 |
| **확장 가능성** | 일반 사용자 대상 확장이 어려움 |
| **원래 목적** | 포트폴리오용 → 이제는 **실질적 제품** 개발 욕구 |
